# Code Style

## 注释规则

为保证项目在持续迭代过程中仍然具备良好的可读性和可维护性，当前仓库统一执行以下注释规范。

### 1. 实体类必须编写具体注释

适用范围：

- JPA 实体类
- 前端用于承载业务数据的数据结构定义
- 与领域模型直接相关的 DTO / 响应模型

要求：

- 说明该类表示什么业务对象
- 说明它在系统中的职责
- 如有必要，说明与数据库表、缓存结构或接口协议的对应关系

示例：

- 合格：`表示一条持久化的待办事项记录，映射数据库中的 todo_item 表。`
- 不合格：`Todo 实体类。`

### 2. 实现方法必须编写具体注释

适用范围：

- 后端 Service、Controller、Config、ExceptionHandler 中的具体方法
- 前端 Vue / TypeScript 中承担实际业务逻辑的函数
- 有副作用、状态变化、缓存更新、参数校验、异常转换等行为的方法

要求：

- 说明方法做了什么
- 必要时说明关键参数含义
- 必要时说明返回值语义
- 如方法包含副作用，需要说明会影响什么状态、缓存或外部资源

示例：

- 合格：`根据主键查询单个待办事项，优先读取对象缓存，缓存未命中时再访问数据库并回填缓存。`
- 不合格：`查询待办事项。`

### 3. 注释不能只重复代码字面意思

禁止写法：

- `获取标题`
- `设置状态`
- `处理异常`
- `刷新数据`

如果注释只是把方法名翻译成中文，而没有提供额外上下文，则视为不合格。

### 4. 注释风格要求

后端 Java：

- 类、方法优先使用 **Javadoc**
- 在需要时补充 `@param`、`@return`
- 注释应简洁明确，避免大段无信息量描述

前端 Vue / TypeScript：

- 接口定义、业务函数使用清晰的块注释
- 重点说明数据结构用途、函数职责和副作用

### 5. 推荐关注的信息

写注释时，优先说明以下内容：

- 这个类 / 方法为什么存在
- 它处理了什么业务约束
- 它与数据库、缓存、接口协议之间的关系
- 它成功或失败时会影响什么状态

### 6. 执行要求

后续新增代码时，默认必须遵守本规范：

- 新增实体类时，同步补类注释
- 新增实现方法时，同步补方法注释
- 前端新增或修改菜单、按钮、图标说明、标题、提示文案等用户可见文本时，必须同步实现国际化
- 前端国际化至少需要提供中文和英文两套文案，禁止将中文或英文直接硬编码在页面组件中长期保留
- 图标如果带有文本标签、tooltip、aria-label、title 或其他可见/可读描述，也必须接入国际化资源统一管理
- 修改已有逻辑时，如原注释已失真，必须一并更新

该规则为当前仓库的长期约束，不作为可选建议处理。

### 7. 前端国际化规则

为保证项目后续可扩展到多语言场景，前端界面的用户可见内容必须默认按照国际化方式设计与实现。

适用范围：

- 导航菜单
- 按钮文本
- 页面标题
- 表单标签
- 占位提示
- 空状态文案
- 成功/失败提示
- 图标对应的文字说明、tooltip、title、aria-label

要求：

- 默认至少支持 `zh-CN` 与 `en`
- 文案应统一收敛到国际化资源文件管理
- 页面组件中禁止长期硬编码中英文展示文案
- 新增前端交互元素时，国际化资源必须与功能代码同步提交

示例：

- 合格：按钮文案通过 `t('todo.actions.create')` 读取，并在中文、英文语言包中都存在对应翻译
- 不合格：在 Vue 模板中直接写死 `新建待办`、`Create Todo` 之类的展示文本

### 8. 方案文档归档规则

为保证产品方案、技术方案和实施计划在项目迭代过程中具备可追溯性，所有新增方案必须归档到项目根目录下的 `docs/` 目录中统一管理。

适用范围：

- 产品路线图
- 功能设计方案
- 接口设计方案
- 数据库设计草案
- 分阶段实施计划
- 重要方案变更说明

要求：

- 每次形成新的正式方案时，必须同步在 `docs/` 目录下新增或更新归档文件
- 方案文件命名应体现主题与版本，例如：`phase-3-plan-v1.md`
- 当已有方案发生实质性变更时，不直接覆盖原方案文件，必须以新版本文件迭代归档，例如从 `v1` 增加为 `v2`
- 新版本方案应明确说明相对上一版本的变化点
- 对外输出给用户的正式方案内容，应以 `docs/` 中已归档的版本为准

示例：

- 合格：本次输出了 Phase 3 方案，同时新增 `docs/phase-3-plan-v1.md` 归档；后续如果方案调整，再新增 `docs/phase-3-plan-v2.md`
- 不合格：在对话中给出方案，但项目内没有任何归档文件；或直接覆盖旧版本导致历史不可追踪
