## 2. 最终交付能力清单

### 2.1 Sprint 3.1：子任务 / Checklist

已完成能力：

- 子任务新增、更新、删除、状态切换
- 主任务下的 checklist 展开/收起
- 子任务完成进度汇总
- 主任务与子任务之间的缓存一致性处理

交付结果：

- 后端支持子任务 CRUD 与汇总
- 前端支持 checklist 面板展示和交互
- 已补 service / controller / 前端测试

### 2.2 Sprint 3.2：重复任务 / Recurrence

已完成能力：

- 支持 `NONE` / `DAILY` / `WEEKLY` / `MONTHLY`
- 支持重复间隔与结束时间
- 支持 `completedAt`
- 支持 `nextTriggerTime`
- 当前执行模型为：**完成当前任务时生成下一条实例**

交付结果：

- 后端完成重复任务校验、实例生成与缓存处理
- 前端支持重复任务创建、编辑、展示
- 文案与标签显示已统一修正

### 2.3 Sprint 3.3：统计面板 / Stats Panel

已完成能力：

- 概览统计
  - 今日完成数
  - 本周完成数
  - 逾期任务数
  - 活动任务数
- 分类统计
  - 分类活动数
  - 分类完成数
  - 无分类归并展示
- 最近 7 天完成趋势

交付结果：

- 后端新增 stats 接口
- 前端新增统计面板并接入 `TodoList`
- 仅在 `ACTIVE` 模式展示统计面板

### 2.4 Sprint 3.4：看板视图 / Static Kanban View

已完成能力：

- 列表视图 / 看板视图切换
- 看板视图首版静态分列
- 仅在 `ACTIVE` 模式开放看板视图
- 回收站模式自动退回列表视图

当前列定义：

- 待办列：`status != DONE`
- 已完成列：`status = DONE`

交付结果：

- 未引入拖拽
- 未引入后端变更
- 复用现有 `TodoItemRow` 和既有交互处理链路

## 5. 已知非阻塞遗留项

以下问题不阻塞 Phase 3 验收，但建议进入后续 backlog：

1. 统计面板加载失败时当前为静默降级，无用户提示
2. 统计趋势柱条对 0 值日期存在极细最小高度展示
3. 看板行为目前主要由 `TodoList.test.ts` 覆盖，缺少 `TodoKanbanView` 独立单测
4. 看板列定义目前为 `DONE / 非 DONE`，若未来状态扩展需重构列分组逻辑
